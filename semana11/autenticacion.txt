El login en una aplicación permite identificar al usuario antes de darle acceso a ciertas funciones.
Esto ayuda a controlar quién puede entrar al sistema y qué acciones puede realizar.
Algunas rutas deben estar protegidas porque pueden contener información privada o registros internos.
También pueden permitir acciones que no deben estar disponibles para cualquier persona.
No se deben exponer públicamente contraseńas, documentos de identidad, teléfonos, direcciones, tokens, claves de API ni datos sensibles de la comunidad.
